# 联调示例

下面给出 3 个真实财政问题，方便你逐步联调整条链路。

## 示例 1：单值汇总查询

问题：

```text
2025年10月一般公共预算收入中税收收入合计是多少？
```

预期链路：

1. 槽位提取
   - 时间：`2025年10月`
   - 预算口径：`一般公共预算收入`
   - 科目：`税收收入合计`
   - 查询类型：`summary`

2. 选表
   - 倾向命中：一般公共预算收入完成情况相关表

3. 指标匹配
   - 倾向命中：金额类指标，如本月金额或累计金额

4. SQL
   - 结果应该是一行一值

5. 页面展示
   - 重点看 `selected_table`、`subject_match`、`metric_match`、`sql`

## 示例 2：趋势查询

问题：

```text
2025年1月到10月全省教育支出每月变化趋势怎么样？
```

预期链路：

1. 槽位提取
   - 时间：`2025年1月-10月`
   - 地区：`全省`
   - 科目：`教育支出`
   - 查询类型：`trend`
   - 图表：`line`

2. 选表
   - 倾向命中：一般公共预算支出完成情况相关表

3. SQL
   - 需要按 `YEAR_MONTH` 分组

4. 图表
   - 期望返回折线图配置

## 示例 3：占比查询

问题：

```text
2025年10月省本级政府性基金收入各科目占比情况是什么？
```

预期链路：

1. 槽位提取
   - 时间：`2025年10月`
   - 地区：`省本级`
   - 预算口径：`政府性基金收入`
   - 查询类型：`proportion`
   - 图表：`pie`

2. 选表
   - 倾向命中：政府性基金收入相关表

3. SQL
   - 需要按科目名称分组

4. 图表
   - 期望返回饼图配置

## 联调建议顺序

1. 先跑 `python quick_check.py`
2. 再启动 `python main.py`
3. 先测“示例1”，确认最短路径跑通
4. 再测“示例2”，确认时间分组和折线图
5. 最后测“示例3”，确认分组统计和饼图

## 每次联调重点看什么

- `intent`
  - 槽位是否提对
- `selected_table`
  - 表是否合理
- `subject_match`
  - 科目是否对
- `metric_match`
  - 指标列是否对
- `sql`
  - where / group by 是否合理
- `rows`
  - 是否有返回
- `chart`
  - 图表类型是否符合问题类型

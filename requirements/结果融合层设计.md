# RAG 反哺 Text2SQL 模块 — 输入输出设计

## 1. 模块定位

在 AIAssistant 框架的请求生命周期中，本模块位于 `request_preprocessing` 之后，替代当前的 `unified_intent_extractor` + `dispatch_execution` 的简单串联。将"提取→路由→各自执行"改为"提取→RAG检索→反哺→增强后重查"：

```
problem_intake → request_preprocessing → ★ RAG反哺Text2SQL模块 ★ → 最终回答
```

**职责**：接收用户自然语言问题，内部完成意图提取、RAG 检索、槽位反哺补全、text2sql 二次查询，输出最终结构化回答。

**核心原则**：RAG 检索到的文档是 text2sql 的"知识外挂"。用户说得模糊→RAG 文档写得更清楚→从文档中提取槽位→帮 text2sql 查得更准。纯 RAG 问题（无查数意图）不进反哺层，RAG 直接回答。

## 3. 模块输出

| 输出项 | 类型 | 说明 |
|--------|------|------|
| `answer` | `str` | 最终自然语言回答（markdown 格式，含表格、引用） |
| `answer_mode` | `str` | 回答类型：sql / rag / both / clarify / fallback |
| `sql_data` | `List[dict] \| None` | SQL 查询原始结果行，供前端图表渲染 |
| `chart_config` | `dict \| None` | ECharts option |
| `citations` | `List[dict]` | 引用来源，每项含 type(sql/rag)、label、detail |
| `enrichment_log` | `dict \| None` | 本次反哺增强了什么：{filled_slots, candidates, sources} |

| answer_mode | 含义 | answer 内容 |
|-------------|------|------------|
| `sql` | 纯查数，SQL精确结果 | 表格 + 数值摘要 |

| `fallback` | 无业务意图 | 兜底话术 |

## 4. 内部处理流程

模块内部包含 4 个子步骤，输入用户问题，输出最终回答：

```
question + history_messages + region_code
                    │
                    ▼
    ┌───────────────────────────────┐
    │  子步骤1: 统一意图提取          │
    │  复用 UnifiedIntentExtractor   │
    │  输入: question, history        │
    │  输出: UnifiedIntentDict (v1)  │
    └───────────────┬───────────────┘
                    │
                    ▼
    ┌────── 守卫 ──────┐
    │ subjects/metrics   │
    │ 或 need_* 非空？   │
    └──┬────────────┬───┘
    否 │            │ 是
       ▼            ▼
   兜底回答      ┌───────────────────────────────┐
                 │  子步骤2: RAG 检索              │
                 │  复用 RagService.search()       │
                 │  输入: intent_v1.rag            │
                 │  输出: RagResult (snippets)     │
                 └───────────────┬───────────────┘
                                 │
                                 ▼
                 ┌──── text2sql.subjects ────┐
                 │  或 metrics 非空？          │
                 └──┬────────────────────┬───┘
                 否 │                    │ 是
                    ▼                    ▼
              纯RAG回答       ┌───────────────────────────────┐
                              │  子步骤3: RAG 反哺增强          │
                              │  从 snippets 提取槽位候选值      │
                              │  投票 → 补全 → intent_v2       │
                              │  输入: intent_v1, snippets      │
                              │  输出: intent_v2, enrichment_log│
                              └───────────────┬───────────────┘
                                              │
                                              ▼
                              ┌───────────────────────────────┐
                              │  子步骤4: 组装回答              │
                              │  槽位齐全 → 执行 text2sql       │
                              │  槽位不全 → clarify回答         │
                              │  输入: intent_v2, rag_result    │
                              │  输出: 最终回答                  │
                              └───────────────────────────────┘
```

核心实现类：

```python
class RagDrivenText2SqlPipeline:
    """RAG 反哺 Text2SQL 管线 — 模块总入口。

    输入:
        question: str                 # 用户问题
        history_messages: List[dict]   # 历史消息（可选）
        region_code: str              # 地区权限码（可选）

    输出:
        answer: str                   # 最终回答（markdown）
        answer_mode: str              # sql / rag / both / clarify / fallback
        sql_data: List[dict] | None
        chart_config: dict | None
        citations: List[dict]
        enrichment_log: dict | None
        overall_confidence: float
    """

    def __init__(self, extractor, rag_service, enricher, text2sql_pipeline):
        self._extractor = extractor
        self._rag_service = rag_service
        self._enricher = enricher
        self._text2sql = text2sql_pipeline

    def run(
        self,
        question: str,
        history_messages: Optional[List[dict]] = None,
        region_code: Optional[str] = None,
    ) -> dict:
        # 步骤1: 提取第一版意图
        intent_v1 = self._extractor.extract(question, history_messages, region_code)

        # 守卫: 无业务意图 → 兜底
        t2s = intent_v1.text2sql
        rag = intent_v1.rag
        if not self._has_business_intent(intent_v1):
            return self._fallback()

        # 步骤2: RAG 始终检索
        rag_result = self._rag_service.search(rag)

        # 纯 RAG 问题（没有 subjects 也没有 metrics）
        if not t2s.subjects and not t2s.metrics:
            return self._rag_only(rag_result)

        # 步骤3: RAG 反哺增强
        intent_v2, log = self._enricher.enrich(intent_v1, rag_result.snippets, question)

        # 步骤4: 组装回答
        return self._build_final_answer(intent_v2, rag_result, log)
```

## 5. 子步骤3 详细设计：RAG 反哺增强

### 5.1 输入

| 输入 | 类型 | 说明 |
|------|------|------|
| `intent_v1` | `UnifiedIntentDict` | 第一版意图 |
| `snippets` | `List[KnowledgeSnippet]` | RAG 检索到的文档片段 |
| `user_question` | `str` | 原始问题 |

### 5.2 输出

| 输出 | 类型 | 说明 |
|------|------|------|
| `intent_v2` | `UnifiedIntentDict` | 增强后的意图（空槽位被补全） |
| `enrichment_log` | `dict` | `{filled_slots, candidates, sources}` |

### 5.3 可补全的槽位

| 槽位 | 类型 | 提取方式 |
|------|------|---------|
| `metrics` | `List[str]` | 指标别名匹配（"执行金额"→"本月金额"） |
| `region_level` | `str` | 关键词：全省/省本级/各市/各区县 |
| `time_text` | `str` | 正则 `\d{4}年` |
| `time_start` | `str` | 从 time_text 计算，全年→yyyymm01 |
| `time_end` | `str` | 从 time_text 计算，全年→yyyymm12 |
| `account_book` | `str` | 关键词：一般公共预算/政府性基金/国有资本经营预算/社会保险基金 |
| `data_stage` | `str` | 关键词：执行数/预算数/草案数/完成情况 |
| `business_module` | `str` | 关键词：预算执行/决算/预算调整/预算草案 |
| `subjects` | `List[str]` | 从文档补全科目全称 |

### 5.4 核心实现

```python
class RagSlotEnricher:
    """RAG 槽位增强器。"""

    def __init__(self, metric_aliases: Dict[str, List[str]]):
        self._metric_aliases = metric_aliases

    def enrich(
        self,
        intent_v1: UnifiedIntentDict,
        snippets: List[KnowledgeSnippet],
        user_question: str,
    ) -> tuple[UnifiedIntentDict, dict]:
        """RAG 反哺主入口。"""
        # [A] 提取候选值
        candidates = self._extract_candidates(snippets)
        # [B] 投票
        voted = self._vote(candidates)
        # [C] 补全
        intent_v2, log = self._fill(intent_v1, voted, candidates)
        return intent_v2, log

    def _extract_candidates(self, snippets) -> Dict[str, List[str]]:
        """从所有 snippet 中提取槽位候选值。"""
        result = defaultdict(list)
        for s in snippets:
            text = s.content
            # 时间
            for m in re.finditer(r'(\d{4})年', text):
                result['time'].append(m.group(0))
            # 地区层级
            for kw in ['全省', '省本级', '各市', '各区县']:
                if kw in text:
                    result['region_level'].append(kw)
            # 指标
            for canonical, aliases in self._metric_aliases.items():
                if any(a in text for a in aliases):
                    result['metrics'].append(canonical)
            # 四本账
            for kw in ['一般公共预算', '政府性基金', '国有资本经营预算', '社会保险基金']:
                if kw in text:
                    result['account_book'].append(kw)
            # 数据阶段、业务模块...同理
        return result

    def _vote(self, candidates) -> Dict[str, tuple]:
        """对每个槽位统计频次，频次≥2才采纳。"""
        result = {}
        for slot, values in candidates.items():
            counter = Counter(values)
            top_val, top_cnt = counter.most_common(1)[0]
            result[slot] = (top_val, 0.9 if top_cnt >= 2 else 0.5)
        return result

    def _fill(self, intent_v1, voted, candidates) -> tuple[UnifiedIntentDict, dict]:
        """只补空字段，非空保持原值。"""
        t2s = copy.deepcopy(intent_v1.text2sql)
        filled = []
        mapping = {
            'metrics': (t2s.metrics, lambda v: [v]),
            'region_level': (t2s.region_level, lambda v: v),
            'time': (t2s.time_text, lambda v: v),
            'account_book': (t2s.account_book, lambda v: v),
            'data_stage': (t2s.data_stage, lambda v: v),
        }
        for slot, (field, transform) in mapping.items():
            if not field and slot in voted:
                val, conf = voted[slot]
                if conf >= 0.5:
                    new_val = transform(val)
                    if slot == 'metrics': t2s.metrics = new_val
                    elif slot == 'region_level': t2s.region_level = new_val
                    elif slot == 'time':
                        t2s.time_text = new_val
                        t2s.time_start, t2s.time_end = self._calc_time(new_val)
                    elif slot == 'account_book': t2s.account_book = new_val
                    elif slot == 'data_stage': t2s.data_stage = new_val
                    filled.append(slot)
        return (
            UnifiedIntentDict(text2sql=t2s, rag=intent_v1.rag),
            {"filled_slots": filled, "candidates": dict(candidates), "sources": {}}
        )
```

## 7. 设计审查

### 7.1 架构问题

#### 问题1：串行依赖导致不必要的延迟

当前流程是严格串行的：`extract → RAG → enrich → text2sql`。即使意图 v1 的槽位已经齐全（如"2025年省本级卫生健康支出的执行金额是多少"），text2sql 也必须等 RAG 检索完成才能执行。总延迟 = extract + RAG + enrich + text2sql。

**修正**：当 v1 槽位齐全时，text2sql 和 RAG 应该并行执行。只有槽位不全时才需要等 RAG 结果做反哺。

#### 问题2：RAG 失败路径未处理

`rag_service.search()` 可能返回空 snippets、检索异常、或全部低相关度片段。当前设计没有处理这些失败路径。如果 RAG 什么都搜不到，enrichment 拿不到候选值，槽位仍然不全，应该走向 clarify。

**修正**：增加 RAG 结果质量检查——snippets 为空或最高相关度 < 阈值时，跳过反哺，直接用 v1 的意图。

#### 问题3：反哺可能引入错误信息

RAG 文档可能涉及不同年份、不同地区的财政数据。比如用户想知道 2025 年的情况，但 RAG 搜到了 2019 年的文档。如果文档里写了"2019年全省…"，enricher 会把 `time` 补全为"2019年"——这是错的。

**修正**：候选提取时必须与原始问题交叉校验——如果用户问题里已经有"2025"字样，RAG 提取的"2019"就不能覆盖。时间槽位的补全优先级应该设为"年份越接近问题中提到的年份，权重越高"。

#### 问题4：`both` 模式在流程图中缺失

输出定义了 `answer_mode=both`，但内部流程图里没有对应的处理逻辑。both 应该在 text2sql 执行成功且 RAG 也有相关解释片段时触发，最终回答同时包含数据表格和文档解读。

**修正**：在步骤4中增加 both 路径。

#### 问题5：`subjects` 槽位提取未实现

§5.3 的可补全槽位表列出了 `subjects`，但 `_extract_candidates` 伪代码没有对应的提取逻辑。这会导致用户的模糊科目名（如"义务教育"）无法被 RAG 文档中的规范名（如"教育支出"）替代。

**修正**：补充 subjects 提取逻辑——用已知科目词典 + 正则 `[一-龥]+(收入|支出)` 从文档中提取候

#### 问题6：`sources` 字段始终为空

`enrichment_log.sources` 被硬编码为 `{}`，无法追踪每个补全值来自哪个文档。线上出问题时无法定位"凭什么把地区层级补成了全省"。

**修正**：在候选提取时记录每个值的来源文档名和片段位置。

### 7.2 边缘情况

| 边缘情况 | 影响 | 当前是否处理 |
|---------|------|:----------:|
| RAG 返回 0 个 snippet | enrichment 无候选 → 槽位补不上 | 未处理 |
| RAG 返回的 snippet 全部低相关度(<0.3) | 不可靠的候选值被采纳 | 未处理 |
| 多文档给出冲突的槽位值（全省 vs 省本级，各出现3次） | 投票平局，选谁？ | 未处理 |
| 文档提到多个年份（"2019年…2020年…2025年…"） | 年份候选池混入无关年份 | 未处理 |
| 用户问题已包含明确槽位值 | RAG 候选值与之冲突 | 未处理（只补空字段，但时间/地区可能冲突） |
| 文档内容跨四本账（既有一般公共预算又有政府性基金） | account_book 投票可能选错 | 未处理 |
| region_code 权限过滤后的 RAG 结果为空 | 判定为"权限不足"而非"无数据" | 已有（复用 region_filter 逻辑） |

### 7.3 缺失的设计要素

| 要素 | 说明 |
|------|------|
| 并行策略 | 槽位齐全时 text2sql 和 RAG 应并行，减少延迟 |
| 降级策略 | RAG 失败/超时时，降级为直接用 v1 查 text2sql |
| 相关度阈值 | RAG snippet.score < 0.3 的不参与槽位提取 |
| 槽位冲突解决 | 多文档矛盾时，优先采信与用户问题中已有信息一致的文档 |
| 时间匹配优先级 | RAG 提取的年份应优先选择与问题最接近的 |
| 缓存策略 | 同一问题短时间内重复问，应命中缓存 |
| 超时控制 | 每步应有超时上限，防止单步卡死 |
| 日志与追踪 | enrichment_log 必须记录每个补全值的来源文档 |

## 8. 审查问题解决方案

### 方案1：串行→按需并行

**修改流程**：步骤1 拿到 intent_v1 后，先判断槽位状态再决定执行方式。

```
步骤1: extract → intent_v1
              │
              ▼
     ┌─ 四槽位齐全？ ──┐
     │                  │
    是                  否
     │                  │
     ▼                  ▼
  text2sql + RAG    只执行 RAG
  并行执行           │
     │               ▼
     │          RAG 有结果？
     │           /      \
     │         是        否
     │         │          │
     │         ▼          ▼
     │    反哺增强     clarify
     │    intent_v2    (补不上)
     │         │
     │         ▼
     │   重跑 text2sql
     │         │
     └────┬────┘
          ▼
      组装回答
```

**伪代码变更**：

```python
def run(self, question, history_messages=None, region_code=None):
    intent_v1 = self._extractor.extract(question, history_messages, region_code)
    if not self._has_business_intent(intent_v1):
        return self._fallback()

    # 槽位齐全 → text2sql 和 RAG 并行
    if self._slots_ready(intent_v1.text2sql):
        sql_future = run_async(self._text2sql.run, intent_v1.text2sql)
        rag_result = self._rag_service.search(intent_v1.rag)
        sql_result = sql_future.result(timeout=30)
        return self._build_answer(sql_result, rag_result, intent_v1)

    # 槽位不全 → 只跑 RAG，等反哺
    rag_result = self._rag_service.search(intent_v1.rag)
    if not rag_result.success or not rag_result.snippets:
        return self._clarify(intent_v1)  # RAG 也没结果，补不上

    intent_v2, log = self._enricher.enrich(intent_v1, rag_result.snippets, question)
    if self._slots_ready(intent_v2.text2sql):
        sql_result = self._text2sql.run(intent_v2.text2sql)
        return self._build_answer(sql_result, rag_result, intent_v2, log)
    else:
        return self._clarify(intent_v2, log)  # 反哺后仍不全
```

### 方案2：RAG 失败降级路径

```python
# 在 enrich 之前增加质量检查
def _should_enrich(self, rag_result, intent_v1):
    if not rag_result.success:
        return False, "RAG 检索失败"
    if not rag_result.snippets:
        return False, "RAG 无结果"
    # 最高相关度 < 阈值 → 不采用
    max_score = max(s.score for s in rag_result.snippets)
    if max_score < 0.3:
        return False, f"RAG 最高相关度 {max_score:.2f} < 0.3"
    return True, ""

# 降级策略：不能反哺时，直接用 v1 的意图查（可能槽位不全→clarify）
can_enrich, reason = self._should_enrich(rag_result, intent_v1)
if not can_enrich:
    if self._slots_ready(intent_v1.text2sql):
        return self._text2sql_only(intent_v1)
    else:
        return self._clarify(intent_v1, degrade_reason=reason)
```

```python
def _extract_candidates(self, snippets, user_question):
    """提取候选值时，与原始问题交叉校验。"""
    result = defaultdict(list)
    # 先从原始问题中提取已知信息作为"锚点"
    anchors = {
        'time': self._extract_time_from_question(user_question),
        'region_level': [kw for kw in ['全省','省本级','各市','各区县'] if kw in user_question],
    }
    
    for s in snippets:
        if s.score < 0.3:  # 低相关度片段跳过
            continue
        # 时间提取 — 优先选择与原始问题年份最接近的
        years_in_doc = re.findall(r'(\d{4})年', s.content)
        question_years = re.findall(r'(\d{4})年', user_question)
        if question_years:
            # 只保留与问题年份相差 ≤1 年的
            q_year = int(question_years[0])
            years_in_doc = [y for y in years_in_doc if abs(int(y) - q_year) <= 1]
        for y in years_in_doc:
            result['time'].append(f"{y}年")
        # ... 其余槽位同理
    return result
```

| 时间 | "2025年" | "2019年" | 不采纳 |
| 时间 | 没有 | "2025年" | 采纳 |

### 方案4：both 模式路径

```python
def _build_answer(self, sql_result, rag_result, intent, log=None):
    has_sql = sql_result and sql_result.success and sql_result.rows
    has_rag = rag_result and rag_result.success and rag_result.snippets
    
    if has_sql and has_rag:
        return self._both(sql_result, rag_result, intent, log)
    elif has_sql:
        return self._sql_only(sql_result)
    elif has_rag:
        return self._rag_only(rag_result)
    else:
        return self._clarify(intent)

def _both(self, sql, rag, intent, log):
    """both 模式：数据表格 + RAG 解读 + 冲突标注。"""
    answer = []
    # 1. 数据表格
    answer.append(self._format_table(sql.rows))
    # 2. 如果反哺过，标注增强信息
    if log and log.get("filled_slots"):
        answer.append(f"\n> 本次查询通过知识库文档补全了以下条件：{', '.join(log['filled_slots'])}")
    # 3. RAG 文档解读
    rag_text = rag.answer or self._format_snippets(rag.snippets)
    answer.append(f"\n## 政策解读\n{rag_text}")
    # 4. 交叉验证（如果两者都有数值）
    conflict = self._detect_value_conflict(sql.rows, rag.snippets)
    if conflict:
        answer.append(f"\n## 数据一致性说明\n{conflict}")
    return {"answer": "\n".join(answer), "answer_mode": "both", ...}
```

```python
def _extract_subjects_from_snippets(self, snippets, user_question):
    """从 RAG 文档补全科目。"""
    candidates = []
    for s in snippets:
        if s.score < 0.3:
            continue
        # 方法1: 已知科目词典匹配
        for keyword in self._subject_keywords:
            if keyword in s.content and keyword not in user_question:
                candidates.append(keyword)
        # 方法2: 正则匹配 "XX支出/XX收入"
        for m in re.finditer(r'([一-龥]{2,18})(收入|支出)', s.content):
            term = m.group(0)
            if term not in user_question:
                candidates.append(term)
    return candidates
```

### 方案7：投票平局处理

```python
def _vote(self, candidates):
    result = {}
    for slot, values in candidates.items():
        counter = Counter(values)
        if not counter:
            continue
        # 取前两名
        top = counter.most_common(2)
        if len(top) == 1:
            result[slot] = (top[0][0], 0.9 if top[0][1] >= 2 else 0.5)
        elif top[0][1] > top[1][1]:  # 有明显胜出者
            result[slot] = (top[0][0], 0.9 if top[0][1] >= 2 else 0.5)
        else:
            # 平局 → 不补全，保持原值（宁可少补，不可错补）
            result[slot] = (None, 0.0)
    return result
```
